# 雷轟（Raigo） UI構成・画面要素定義書

本ドキュメントは、ゲーム『雷轟（Raigo）』のボードゲームアリーナ（BGA）における画面構成および各要素の定義をまとめたものである。今後の設計・改修の基準資料として使用する。

## 1. 盤面の基礎構造
盤面は中央線を境に上下対照（プレイヤーA：相手、プレイヤーB：自分）の構成となっている。

- **背景:** 木目調のテーブル。
- **中央線 (`raigo-center-line`):** 盤面を上下に分ける水平な境界線。

## 2. 共通領域（中央エリア）
全プレイヤーが共通で使用、または物理的に中央に配置されるコンテナ。

- **峡谷 (`kyoukokuA`, `kyoukokuB`):** 
  - **役割:** 捨て駒、または効果によって公開された駒の置き場。
  - **配置:** 中央線のすぐ上下。各24個分のスペース。
- **雷山 / 山札 (`deckA`, `deckB`):**
  - **役割:** 山札から駒を引くための領域。
  - **配置:** 峡谷の外側。盤面で最も横に長い矩形。

## 3. 建設領域（サイドエリア）
塔を建設するための領域。

- **塔列 (`towerA1-7`, `towerB1-7`):**
  - **役割:** 駒を垂直に積み上げ、「役」を完成させる場所。
  - **配置:** 盤面の左右両端。計14列（片側7列ずつ）。

## 4. プレイマット（プレイヤー個別領域）
各プレイヤーの手手前で、駒の管理や準備を行う領域。

- **桜門 (`oumonA1-3`, `oumonB1-3`):**
  - **役割:** 塔を建てるための基点となる3つの円形領域。
- **体内 (`insideA1-3`, `insideB1-3`):**
  - **役割:** 非公開の手札を置く3つの正方形コンテナ。
- **手 (`handA1-2`, `handB1-2`):**
  - **役割:** アクション（移動、発動など）に使用する駒を一時的に置く2つの正方形コンテナ。
- **月 (`moonA`, `moonB`):**
  - **役割:** 特殊な効果や特定の状況で使用される独立した正方形コンテナ。

## 5. UI パネル
ゲームの進行状況を表示・制御する要素。

- **フェーズパネル (`raigo-phase-panel`):**
  - **役割:** 現在のフェーズ名（「開」「現」「選」「積/発」「隠」）を表示。
  - **動作:** 手番プレイヤーの側に移動し、アクティブ状態（オレンジ色のグラデーション）になる。

## 6. 駒 (Pieces) の定義
駒は漢字が記された正方形で、以下の3種に大別される。

| カテゴリ | 主な駒 | 役割 |
| :--- | :--- | :--- |
| **音無 (数駒)** | 一、二、三、四、五、六、七 | 基本的な塔の構成要素。数字を持つ。 |
| **言霊 (ことだま)** | 雷、蛇、斬、陣、轟、霧、瞬、浄 | 発動時に特殊な効果（攻撃、防御、操作など）を持つ。 |
| **奥義** | 鳳、鬼、極、楼、舞、嵐、神など | 非常に強力な効果を持つ特殊駒。戦局を大きく左右する。 |

## 7. 実装上の注意点
- **回転:** プレイヤー2以降の場合、盤面全体が180度回転（`board-rotated`）して表示される。
- **駒の向き:** 上側（相手プレイヤー側）のコンテナにある駒は、相手から読みやすいように180度回転して表示される。
- **ハイライト:** フェーズやアクションに応じて、対象のコンテナや駒が黄緑色（`available-hand`）などにハイライトされる。
